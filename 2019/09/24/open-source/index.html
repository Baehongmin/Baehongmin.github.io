오픈소스 SW의 개요

  • 등장배경 - 공개소프트웨어 개념 - SW가 HW인 컴퓨터와 분리되어 상품화 되어가는 과정에서 반작용적으로 등장

  • Ope Source Initiative(OSI, https://opensource.org/) - 10가지 기준

  • 오픈소스SW 라이센스 종합정보시스템 - Docker …

  • 국내 회사 - 삼성전자 - AVAS 2018 오픈소스 취약점 자동검출시스템 - 엘지전자 - 한국전력연구원 - SWAT : 전력소프트웨어 검사 종합 시스템 - SDLC에서 개발 또는 유지보수중 OSS 검사

  • 오픈 SW 거버넌스 - 정의 - 한국정보통신기술협회 : 공공 공개 소프트웨어 거버넌스 지침 - 오픈소스 정책 및 절차 관리 –> 표준기술 적용 및 신규버전 검증 –> 소스코드 검증을 통한 라이선스 관리 –> 보안 취약점 점검 및 예방 –> 정보검색 및 제공

  • 오픈소스 SW 라이센스 - MIT, Apache, AGPL, MPL - 저작권, 특허권, 라이센스, 컴플라이언스

  • 오픈소스 SW 분쟁 사례 - 한컴 MDS - 한컴 Ghostscript 2013년 이후 한번도 공개한적이 없고 비용도 지급하지않음 - Ghostscript는 AFPL과 GPL을 따르는 듀얼 라이센스 정책을 따르고있다. - 벌금 지불함

  • 오픈소스 SW 라이센스의 고지 - 도움말, 설치안내서, 고객센터/도움말 –> 특정 규칙이나 방법이 없음

  • 오픈소스 SW 보안취약점 - WhiteSource 에서 발표 - 버퍼 오버플로우, 중간에 악성코드 삽입 - 아파치 스트럿츠2 취약점(개인정보 유출)

  • 보안 및 품질 문제 해결 비용 - 코딩하는 부분에서 문제가 발견되면 완성됬을때에 비해서 해결 비용이 적게 든다. -

오후 실습

  • 오픈소스 관리 요소 - 소프트웨어의 오픈소스 구성 목록화 - 보안 취약점 탐지 및 업데이트 필요 - 종합적인 라이센스 위험 평가 - 오픈소스 버그 및 품질 평가

  • PCI-DSS Open source - 금융권에서 사용하는 Open source 정책

  • WhiteSource : 오픈소스 관리 자동화 - Synopsys 오픈소스 관리는 라이센스만 해줌 최근에 보안취약점 등을 확인할 수있도록 디지털 시그니처 방식으로 바뀜

  • WhiteSource Chrome 확장 도구로 제공됨 - 다운받을때 부터 정책위반등 정보 확인가능
