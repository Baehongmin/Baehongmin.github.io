Project Euler

Problem 7

소수를 크기 순으로 나열하면 2, 3, 5, 7, 11, 13, ... 과 같이 됩니다.

이 때 10,001번째의 소수를 구하세요.

해답

//Euler 7번

//10001번째 소수

// ArrayList로 구현

ArrayList<Boolean> primeList;

int n = 1000000;

// n+1만큼 할당

primeList = new ArrayList<Boolean>(n+1);

// 0번째와 1번째를 소수 아님으로 처리

primeList.add(false);

primeList.add(false);

// 2~ n 까지 소수로 설정

for(int i=2; i<=n; i++ )

primeList.add(i, true);

// 2 부터 ~ i*i <= n

// 각각의 배수들을 지워간다.

for(int i=2; (i*i)<=n; i++){

if(primeList.get(i)){

for(int j = i*i; j<=n; j+=i) {

primeList.set(j, false);    }    
}        }

int num = 1;

for(int count = 0; count < primeList.size();count++) {

if(primeList.get(count)) {

if(num == 10001) {

System.out.println(num+ " 번쨰: " + count);

break;    }    
num++;
}        }

에라토스테네스의 채를 사용하여 풀어봤습니다. 그냥 n을 100000까지 생성해서 거기안에서 10001 번째 소수를 찾는걸 해보았습니다.
소수가 더 증가하면 증가할수록 몇까지 메모리를 할당해야할지… 전혀 모릅니다.
코드를 수정해서 10001번째 소수를 찾을 수 있도록 하는 프로그램으로 바꿀 예정입니다.

후기

소수를 구하는 방법에 대해서 자기자신 과 1밖에는 나눠지지 않는 방법으로 푸른게 알고리즘적으로 더 좋은 방법이 아닐 거라고 생각하지만 n번째 소수를 구하는 문제에 관해서는 순차적으로 가는게 더 빠를 수도 있지않을까 생각해보지만 혼자만 고민합니다… 핳핳

Project Euler

Problem 6

1부터 10까지 자연수를 각각 제곱해 더하면 다음과 같습니다 (제곱의 합).

> 12  + 22  + ... + 102  = 385

1부터 10을 먼저 더한 다음에 그 결과를 제곱하면 다음과 같습니다 (합의 제곱).

> (1 + 2 + ... + 10)2  = 552  = 3025

따라서 1부터 10까지 자연수에 대해 "합의 제곱"과 "제곱의 합" 의 차이는 3025 - 385 = 2640 이 됩니다.

그러면 1부터 100까지 자연수에 대해 "합의 제곱"과 "제곱의 합"의 차이는 얼마입니까?

해답

//project Euler No.6

int n = 100;

int sum = n*(n+1)/2;

int mul = n*(n+1)*(2*n+1)/6;

System.out.println(sum*sum-mul);

등차 수열의 a항 부터 b항까지의 합은 b(b+a)/2 이므로 합의 제곱은 위의 sum 을 2번 곱하게 되면 합의 제곱의 값을 손쉽게 알 수 있습니다.

또한 제곱의 합은 https://mathbang.net/628 이곳을 참고하여 풀었습니다. 고등 수학의 기억이 새록 새록 떠올랐습니다.

후기

그냥 1~100까지 하는 방법도 있었지만 공식을 사용하는 편이 알고리즘의 효율성을 증가 시킬 수 있는 부분이라고 생각했습니다. for문을 전혀 쓰지 않아도 답이 나옵니다.

Monstache 사용법

프로젝트 중에 MongoDB Data를 ElasticSearch 로 옮겨야 되는 상황이 발생하여 좋은 Plugin 및 Logstash 등을 찾고 있었습니다. 실시간으로 데이터가 이관되는 시스템은 2개이상의 프로그램 연계를 하거나 oplog를 사용하여 Queue를 만들어 사용해야 했기 때문에 번거롭기도하고, 안정적이지 않기 때문에 이 부분에 있어서 이미 꽤 안정적으로 작동되는 프로그램인 Monstache 를 사용하도록 하였습니다.

환경

–> Elastic 6.5.4
–> MongoDB 4.0.9
–> Monstache 5.0.6

Setting

Monstache –> Setting.toml, Pre-Build Monstache 실행 파일, tov_plugin.so (which is compiled same environment of Monstache’s)

문제점

  1. Go lang build
    몽고디비에서 엘라스틱으로 옮길때 약간의 데이터 변동이 필요합니다. 때문에 Go Language 를 사용하여 데이터를 변조하려고 하는데 빌드가 되지 않아 약간의 검색을 해보니 아래와 같이 환경을 맞춰줘야 한다고 합니다.

    Golang plugins must be built with the exact same source code (including dependencies) of the loading program.

    때문에 tov_plugin.so 파일에 명시된 빌드 환경과 실제 환경이 똑같아야 빌드가 되어 사용이 가능합니다.

  2. MongoDB와 같은 곳에 설치
    몽고디비와 같은 곳에 설치 되지 않은 경우 Setting.toml 에서 설정이 더 필요한것 같다.

Project Euler

어렵네요

문제 5

1 ~ 10 사이의 어떤 수로도 나누어 떨어지는 가장 작은 수는 2520입니다.

그러면 1 ~ 20 사이의 어떤 수로도 나누어 떨어지는 가장 작은 수는 얼마입니까?

풀이

    {
    long gong = 1;
    long u = 0;
    long result = 0;
    for(int k = 2;k<20;k++) {    
        u = uclid(gong, k);
        gong = gong/u * k /u * u;
        result = gong;
    }
    System.out.println(result);
    }
public long uclid(long a, long b) {
    long temp = 0;
    while(a!=0) {
        if(a<b) {
            temp = a;
            a = b;
            b = temp;
        }
        a= a%b;
    }
    return b;
}

이 문제는 공약수들의 곱을 이야기 합니다…
여러수의 공약수를 구하기는 쉽지 않다보니 유클리드 호제법이라는 알고리즘으로 두 개의 숫자의 공약수를 구하는 방식으로 코딩해 보았습니다.

유클리드 호제법을 이용하여 제수를 계속 두 수의 나머지를 이용하여 나누는 방식으로
최대 공배수를 구하는 방법으로 공약수의 곱을 구하는 방법을 택했습니다.
